# Pre-Registration — JAMA Network Open n=100 Audit

Dated: 2026-07-01, written and frozen before any citation-checking began on
this sample.

## Sample frame

- **Journal:** JAMA Network Open (fully open access; chosen over flagship
  JAMA for a clean, uniformly-accessible sampling frame — see
  `audit/NOTES.md` for the tradeoff discussion).
- **Article type:** "Original Investigation" only, per the JATS
  `<subj-group subj-group-type="heading"><subject>` value — this excludes
  Research Letters, Corrections, Editorials, Comments, and Viewpoints.
  (Note: the JATS `article-type` attribute alone cannot make this
  distinction — both Original Investigations and Research Letters carry
  `article-type="research-article"` in this journal's output.)
- **Publication window:** calendar year 2025 (2025-01-01 to 2025-12-31),
  not "most recent" — a live check found only ~27% of the newest articles
  had PMC full text linked yet (deposit lag), vs. consistent availability
  for a year-old window.
- **Minimum quality thresholds:** body text >3000 characters, >=10
  references (excludes stub/short pieces that slipped through the heading
  filter).
- **n = 100**, drawn by seeded random sample (seed 42) from a pool of 429
  qualifying articles found within the search budget (see below).
- **Sampling procedure:** PMC record IDs for the full search window were
  fetched, then shuffled (seed 42) *before* fetching/filtering, so that
  stopping early once a 4x margin over n was reached still constitutes an
  unbiased random sample of the full population — order returned by
  `esearch` is not random (relevance/uid), so filtering in that order and
  stopping early would have biased the sample toward whatever `esearch`
  ranks first. (This was caught and fixed after an initial run — see
  `audit/NOTES.md`.)
- **Frozen sample list:** `audit/data/preregistration/jama_network_open_sample_2025_100.json`
  — one row per article: PMCID, PMID, DOI, title, heading, reference count,
  body length. Generated by `audit/sample_jama_network_open.py`. **This
  file is not to be regenerated after citation-checking begins** — if the
  script needs a bug fix, the existing frozen sample list stays authoritative
  regardless of what a re-run would now produce.

## Retrieval method

Full text and reference lists are retrieved via PMC's structured JATS XML
(`audit/jats_parser.py`), not the PDF-to-Markdown path used for the earlier
3-paper calibration set (Jia 2023, Kang 2022, Karabulut 2020). This
eliminates an entire class of bugs found during calibration (reference-list
boundary contamination, heading-regex mismatches, running-header splicing,
superscript-as-citation false positives) because JATS provides real
document structure instead of flat text that has to be reverse-engineered.
Reference entries in JATS frequently carry the cited paper's DOI directly,
letting `check_claims.py` skip Crossref bibliographic search (`ref_resolver.py`)
entirely for those — smoke tests on 3 sample articles showed 93-100%
reference-DOI availability directly from the XML.

## Primary and secondary outcomes

See `docs/adjudication-criteria.md`, "Primary outcome" section (dated
2026-07-01): confirmed-miscitation rate among flagged (non-SUPPORTED)
citations, adjudicated CONFIRMED / SYNTHESIS / ABSTRACT-LIMITED / REJECTED
per that document's categories. Secondary/descriptive outcomes: overall
flagged rate, FLAG_TYPE breakdown, resolution rate, ABSTRACT-LIMITED rate
(gate-2 rescue frequency).

## Reviewer

Solo review (see `docs/adjudication-criteria.md`, "Second reviewer"
section). No second rater for this run — logged as a limitation to carry
into the eventual manuscript, not glossed over.

## What "run the pipeline" means for this sample

For each of the 100 articles: `check_claims.py --pmcid <id>` (live, not
`--dry-run` — requires `ANTHROPIC_API_KEY`, not yet set in this environment
as of this writing). Every LLM call (gate 1 abstract check, gate 2
full-text check where triggered) is cached by hash and logged per the
existing `logs/llm_calls/` convention. No interactive/manual LLM
substitution for this run — the 3-paper calibration set already served that
purpose (see `audit/results/preview/three_paper_review_summary.md`); this
n=100 run is meant to be the first real, reproducible, automated-pipeline
data.
